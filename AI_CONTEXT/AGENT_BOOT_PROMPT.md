# AGENT_BOOT_PROMPT — 新 agent 啟動 prompt

使用方式：每次開新 agent，先把下方分隔線內的內容原樣貼給它，
再貼任務單（照 `TASK_TEMPLATE.md` 填好的）。

---

你即將接手 jonaminz 專案的一個任務。開始任何工作之前，先完成以下啟動程序：

## 啟動程序（依序執行，做完才能動工）

1. **完整閱讀 `AI_CONTEXT/` 內全部七份文件**，依這個順序：
   - `RULES.md` — 禁止與允許事項（最優先，違反任何一條都算任務失敗）
   - `PROJECT_STATE.md` — 專案現況、已完成/未完成、UNKNOWN 清單
   - `ARCHITECTURE.md` — 分層、資料流、設定流、部署流
   - `ACCEPTANCE.md` — 你完工前必須跑的通用驗收
   - `CHANGELOG.md` — 最近幾筆，了解上一棒做到哪
   - `TASK_TEMPLATE.md` — 對照你即將收到的任務單格式與衝突處理規則
   - `AGENT_BOOT_PROMPT.md` — 即本文件，讀到這裡已完成
2. 讀完後，用 3-5 行複述：這個專案是什麼、本次任務要做什麼、
   哪些檔案你被允許修改。複述有誤會被糾正，這比做錯了才發現便宜得多。

## 行為規範

- **不要依賴聊天記憶或你自己的既有印象。** 一切以 `AI_CONTEXT/` 文件與
  repo 現況為準。文件與程式碼不符時，以程式碼為準，並回報文件過時之處。
- **不要腦補。** 文件標記 `UNKNOWN` 或 `NEEDS_CONFIRMATION` 的地方就是
  不確定，不要當成事實使用；需要那個資訊才能完成任務時，先問。
- **不要修改任務單白名單以外的任何檔案。** 包括「順手修正」「順手重構」
  「順手整理格式」——都不行。發現值得修的東西，寫進回報的建議欄。
- **修改前先列出計畫**：要改哪些檔案、每個檔案改什麼、為什麼。
  計畫與任務單有出入時先停下來確認，再動手。
- **修改後回報**：實際變更清單（與計畫比對）、每項變更的理由、
  驗收結果（任務單專屬驗收 + ACCEPTANCE.md 通用驗收，逐條 ✅/❌）、
  未完成事項與原因。
- **收尾**：更新 `AI_CONTEXT/PROJECT_STATE.md`（若狀態有變）、
  在 `AI_CONTEXT/CHANGELOG.md` 最上方追加一筆（格式照該檔模板）。
- 遇到衝突（任務 vs 規則、任務 vs 架構、需要白名單外檔案）：
  **停下來問，附具體衝突點與你的建議選項**，不要自行取捨。
- 對話回覆一律使用繁體中文。
