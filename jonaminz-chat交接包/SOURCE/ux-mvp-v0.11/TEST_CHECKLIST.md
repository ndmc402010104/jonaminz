# Jonaminz Chat + Shared MVP v0.7 測試清單

請先按「重設 Demo」，再依序測試。

## A. 右下角頭像與視窗狀態

- [ ] Jonathan 身分時，右下角顯示 `M`。
- [ ] Minz 身分時，右下角顯示 `J`。
- [ ] 點右下角頭像，開啟半版聊天室。
- [ ] 聊天展開後，右下角頭像仍保留且不變成符號。
- [ ] 再點同一顆頭像，聊天室收合。
- [ ] 標題列沒有 `—`。
- [ ] 標題列沒有 `×`。
- [ ] 點 `⛶`，半版切換成全版。
- [ ] 全版時按鈕變成回到半版的符號。
- [ ] 再按一次，回到半版。
- [ ] 手機寬度下，全版應真正占滿螢幕。

## B. 日常聊天

- [ ] 半版時仍可看到後面的原頁面內容。
- [ ] 可傳送文字。
- [ ] 可用 `＋` 選單傳送目前內容。
- [ ] 可建立截圖卡片示意。
- [ ] 可加入 OneDrive 檔案卡片示意。
- [ ] 可傳送貼圖或愛心。
- [ ] 切換 Jonathan / Minz 後，訊息左右方向與對方名稱正確。

## C. 分享內容與 Seen

- [ ] 預設「你尚未看到」篩選可以按。
- [ ] 「對方尚未看到」可以按。
- [ ] 「尚未一起看到」可以按。
- [ ] 「全部分享」可以按。
- [ ] 未看的卡片有紅點。
- [ ] 卡片只是出現在畫面上，不會自動變已看。
- [ ] 點進卡片詳細內容後，紅點才消失。
- [ ] 關閉詳細內容後，篩選數量同步更新。
- [ ] 詳細內容可標回未看，紅點恢復。
- [ ] 已看的卡片不會整張變灰。
- [ ] 點「討論」後，聊天室顯示正在討論哪一筆分享。
- [ ] 後續文字訊息會顯示回覆該分享。

## D. 電話

- [ ] 右上角電話按鈕可開啟撥號確認。
- [ ] 對方名稱與字母正確。
- [ ] 可測試「直接撥打」→「通話中」→「掛斷」流程。
- [ ] 電話流程不會破壞聊天室狀態。

## E. Shared 去重

- [ ] 在 Shared 貼上一個新網址，可新增內容。
- [ ] 再貼完全相同網址，應合併，不新增第二張卡。
- [ ] 帶 `utm_*` 或 `fbclid` 的同網址，應視為同一筆。
- [ ] 合併後顯示雙方都分享過與分享次數。

## F. Chat Library 概念

- [ ] 頁面可看到目前 instance：`couple-chat`。
- [ ] 頁面可看到未來 `skhps-chat` 的獨立實例示意。
- [ ] 目前聊天室 UI 沒有把 SKHPSv2 的資料或權限寫死進去。

## 最重要的主觀驗收

請直接回答這四題：

1. 半版高度是否剛好，還是太高／太低？
2. 全版切換是否真的有用途？
3. 右下角對方頭像是否比標題列縮小按鈕直覺？
4. Shared 放在完整聊天室中是否自然，還是仍像另一個後台？

## G. v0.8 手機大頭貼與 Messenger 體驗

- [ ] 手機全版為深色沉浸聊天室，不受後方頁面主題影響。
- [ ] Header 顯示對方大頭貼字母與姓名。
- [ ] Header 顯示電話、視訊、資訊與半版／全版按鈕。
- [ ] 對方連續訊息只在該組最後一則顯示頭像。
- [ ] 自己的訊息不重複顯示自己的頭像。
- [ ] 自己最後一則訊息下方顯示對方小頭像，代表已讀位置。
- [ ] 分享內容卡片在訊息流中尺寸自然，不會太像後台卡片。
- [ ] 點資訊按鈕可看到對方資料與電話／視訊／檔案入口。
- [ ] 點視訊可看到 MVP 提示。
- [ ] 半版仍維持原本較輕的 Jonaminz Theme。
- [ ] 全版切回半版後，訊息位置與輸入內容不應遺失。

### 主觀測試

1. 大頭貼是否讓聊天室更像「對方就在這裡」，而不是一般工具視窗？
2. 對方訊息頭像應顯示在每組第一則、最後一則，還是每一則？
3. 已讀小頭像放在自己最後一則訊息下方是否直覺？
4. 手機全版是否需要把「分享內容」Tab 藏進資訊頁，讓頂部更接近 Messenger？

## H. v0.9 操作測試

### Header

- [ ] 右上角只剩電話按鈕。
- [ ] 頂部細橫條可切換半版／全版。
- [ ] 沒有視訊與資訊按鈕。

### Composer

- [ ] 輸入框空白時，最右側顯示 `👍`。
- [ ] 點 `👍` 可直接送出快速心情。
- [ ] 長按 `👍` 可選擇其他符號。
- [ ] 更換後，空白輸入框旁顯示新的預設符號。
- [ ] 輸入文字後，同一顆按鈕自動變成送出。
- [ ] 清空文字後，按鈕恢復為快速心情。
- [ ] 貼圖按鈕位於輸入框右側，點擊可開啟貼圖面板。
- [ ] 選擇貼圖後會以大貼圖訊息送出。

### 訊息心情

- [ ] 長按文字訊息可開啟心情列。
- [ ] 長按分享內容卡片也可開啟心情列。
- [ ] 選擇心情後，符號顯示在訊息下方。
- [ ] 再選相同心情可取消。
- [ ] Jonathan／Minz 可對同一訊息留下不同心情，數量正確合併。

### Shared 卡片

- [ ] 卡片能清楚辨認來源、網域、分享者、標題與附言。
- [ ] 卡片表面沒有固定愛心／笑臉按鈕。
- [ ] 卡片表面沒有固定 Movie／Travel 按鈕。
- [ ] 點 `送往…` 可打開目的地選擇器。
- [ ] 可加入或移除 Movie／Travel。
- [ ] 選擇結果以短摘要顯示，不會讓卡片越來越高。
- [ ] `⋯` 仍可標回未看、開啟原始內容與封存。

### 主觀測試

1. Shared 卡片現在比較像「內容收件匣」，還是仍太像後台？
2. 空白按讚／有字送出的切換是否直覺？
3. 長按訊息加心情是否比把反應按鈕固定顯示更乾淨？
4. 貼圖放在輸入框旁是否符合預期？

- [ ] 切到「全部分享」。
- [ ] 點開任一較下面的內容。
- [ ] 關閉詳細頁後，該內容回到最上面。

- [ ] 點 Header 狀態文字，可模擬對方輸入中。
- [ ] 對方輸入中時，Header 與訊息底部同時顯示 typing indicator。
- [ ] 切換身分並送出新訊息，再收合聊天室。
- [ ] 切回另一人後，右下角 avatar 出現未讀數。
- [ ] 打開聊天室可看到未讀分隔線。
- [ ] 約一秒後自動標為已讀，未讀 badge 消失。
- [ ] 切回發訊息者後，對方小頭像顯示在已讀位置。

### 主觀測試

1. Hover 三顆按鈕是否比固定顯示心情乾淨？
2. 心情列是否貼得太近／太遠？
3. 表情插入文字是否符合 FB 的操作直覺？
4. 「剛看過的 Shared 回到最上面」是否符合你找回內容的習慣？
